# Strategic Decisions Framework

For major decisions that cannot be answered with data alone. Apply when user or stakeholders request strategic recommendations.

## When to Use

Trigger phrases:
- "is it worth opening a store in X?"
- "which is better, A or B?"
- "should we continue or stop [initiative]?"
- "what do you think about [strategic decision]?"

Don't use for: daily operational questions, data requests, tasks with clear answers.

---

## Framework: 5-Layer Analysis

### Layer 1: Clarify the Real Question

Often the question asked is not the question that actually needs to be answered.

```
User: "Is it worth opening a store in location X?"

Real question might be: 
- "Will location X be ROI-positive within 12 months?"
- "Does location X fit our brand positioning?"
- "Do we have operational capacity for expansion?"
```

Confirm first: "Should I analyze from a financial, operational, strategic fit, or comprehensive perspective?"

---

### Layer 2: Second + Third Order Analysis (10/10/10)

For each option, ask:

```
10 MINUTES ahead (immediate):
- What happens immediately if this decision is made?
- What resources are needed right away?
- Who is impacted first?

10 MONTHS ahead (medium-term):
- What's the impact in Q3-Q4 this year?
- Does this open or close other options?
- How does this affect cash flow?

10 YEARS ahead (long-term):
- Is this aligned with long-term direction?
- What's the opportunity cost?
- If it fails, how reversible/recoverable is it?
```

---

### Layer 3: Regret Matrix

Create 2x2 matrix:

```
                     TAKE action     | DON'T take action
Success:           [outcome A]      | [outcome B — missed opportunity]
Failure:           [outcome C]      | [outcome D — dodged bullet]
```

Question: "In which scenario can you best live with the decision?"

Regret minimization: Choose the option where, if it fails, the regret is most acceptable.

---

### Layer 4: Pre-Mortem (Devil's Advocate)

Imagine the decision has been made and failed completely 6 months later.

```
"We opened the new location, and now 6 months later it's losing money. 
Why did this happen?"

Top 5 potential failures:
1. [Most likely reason] → Mitigation: [...]
2. [Second reason] → Mitigation: [...]
3. ...
```

If all failure modes are mitigatable → proceed. If any are unmitigatable → reconsider.

---

### Layer 5: Recommendation

After analysis, deliver concrete recommendations — avoid fence-sitting.

**Format:**
```
🎯 RECOMMENDATION: [Suggested option]

KEY REASONS:
• [Reason 1 — highest weight]
• [Reason 2]
• [Reason 3]

CONDITIONS THAT MUST BE MET:
• [Deal-breaker 1 — if not met, don't proceed]
• [Deal-breaker 2]

FIRST STEPS (if agreed):
• [Concrete action item, who's responsible, when]
```

Always include escape clause: "If [condition X] changes, the recommendation might differ."

---

## Quick Version (for smaller decisions)

If scope is not too large, use:

```
PRO (concrete, data-backed):
• [Pro 1]
• [Pro 2]

CON (concrete, especially non-obvious):
• [Con 1]
• [Con 2]

BOTTOM LINE: [One-sentence recommendation]
```

---

## Notes

- Always acknowledge uncertainty: "This analysis is based on current available data. If [assumption X] changes, the conclusion might differ."
- Don't pretend to have data you don't have (see `anti-hallucination` skill)
- Recommendations must be concrete — avoid "it depends" without explaining what it depends on
